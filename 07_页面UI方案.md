# 页面 UI 方案

本文档基于 `06_App信息架构.md` 继续向下推进，目标是输出首版 App 的页面 UI 设计方向与界面规范。

适用范围：

- 首页
- 闪记弹层
- 回看箱
- 统计页
- 我的页

本文档不等于最终视觉稿，但已经足够支持：

- 线框图设计
- 中保真界面设计
- 组件库初版搭建
- 前端页面拆分

## 1. UI 设计目标

产品的 UI 必须围绕“低打扰、快输入、快返回”展开，因此应满足：

- 视觉安静，不制造新的注意力负担
- 关键信息一眼可见，减少理解成本
- 高频动作触达近、反馈轻、路径短
- 页面之间风格统一，状态清晰

## 2. 视觉气质

建议视觉关键词：

- 轻
- 静
- 柔和
- 克制
- 安全感
- 可回收

避免的视觉方向：

- 过强游戏化
- 高饱和撞色
- 信息爆炸型看板风
- 过度拟物或过度装饰

## 3. 色彩建议

首版建议走“浅背景 + 低饱和功能色”的体系。

### 主色

- 主品牌色：柔和蓝绿色
- 用途：主按钮、高亮状态、重点数据

建议值：

- Primary: `#4C8BF5`
- PrimarySoft: `#EAF2FF`

### 辅助色

- 成功色：`#34C759`
- 警示色：`#FFB340`
- 错误色：`#FF6B6B`
- 中性深灰：`#1F2937`
- 中性正文灰：`#4B5563`
- 中性浅灰：`#E5E7EB`
- 页面底色：`#F7F8FA`
- 卡片白：`#FFFFFF`

### 标签色建议

- 杂念：浅灰蓝
- 待查：浅紫
- 任务：浅蓝
- 情绪：浅橙粉
- 灵感：浅绿

原则：

- 标签色只做辅助识别
- 不要比主按钮更抢眼

## 4. 字体与层级

建议采用系统字体，以保证阅读效率和开发落地性。

### 字号层级

- 页面主标题：24
- 区块标题：18
- 卡片标题：16
- 正文：14
- 辅助说明：12
- 数据大数字：28-32

### 字重建议

- 主标题：Semibold
- 模块标题：Semibold
- 正文：Regular
- 按钮：Medium

### 文案风格

- 尽量短句
- 避免命令式压迫语气
- 偏陪伴式但不过度拟人

推荐文案风格：

- 已记下，先回来吧
- 今天还有 3 条待回看
- 先不展开，稍后处理

## 5. 布局规范

### 页面基础布局

- 顶部安全区留白充足
- 页面左右边距建议 `16-20px`
- 卡片间距建议 `12-16px`
- 模块上下间距建议 `20-24px`

### 圆角与阴影

- 卡片圆角：`16px`
- 按钮圆角：`14-20px`
- 弹层圆角：顶部 `24px`
- 阴影：轻阴影，强调悬浮感但不过重

### 交互热区

- 主要点击区高度不低于 `44px`
- 底部主按钮不小于 `52px`

## 6. 组件规范

### 6.1 主按钮

使用场景：

- `记一下`
- 保存并回去
- 开始专注

样式建议：

- 高度：52
- 圆角：18
- 背景：主品牌色
- 文本：白色，中等字重

### 6.2 次按钮

使用场景：

- 取消
- 延后
- 查看全部

样式建议：

- 透明或浅色底
- 边框可选
- 不应与主按钮争抢视觉优先级

### 6.3 记录卡片

组成建议：

- 内容主文案
- 来源任务
- 时间说明
- 标签
- 快捷动作区

交互建议：

- 默认整卡可点击查看详情
- 支持滑动快捷处理

### 6.4 标签 Chip

用途：

- 标识记录类型
- 快速筛选

样式建议：

- 小号圆角胶囊
- 浅底色 + 深色文字

### 6.5 筛选分段控件

用途：

- 回看箱筛选
- 统计页维度切换

建议：

- 采用 iOS 风格 segmented control
- 状态切换要明确但不过分跳跃

### 6.6 提示条

用途：

- 到期提醒概览
- 轻量反馈

建议：

- 使用浅色底卡片
- 配简短说明和小图标

## 7. 关键页面 UI 方案

## 7.1 首页

### 首页目标

- 让用户一打开 App 就知道“我现在在做什么”
- 让 `记一下` 成为最顺手的动作
- 让待回看内容存在感适中

### 首页布局

```text
顶部问候
当前任务卡片
番茄钟模块
主按钮：记一下
今日待回看卡片
底部导航
```

### 首页视觉重点

首页最强视觉焦点建议只有两个：

- 当前任务卡片
- `记一下` 按钮

其他元素都应退后，避免首页变成效率仪表盘。

### 首页卡片建议

#### 当前任务卡片

建议内容：

- 任务名
- 当前状态
- 已专注时间
- 开始/暂停按钮

样式建议：

- 大卡片
- 白底或浅品牌色底
- 可加一条极浅进度条

#### 今日待回看卡片

建议内容：

- 今日待处理数量
- 最近 2-3 条记录摘要
- 查看全部入口

样式建议：

- 低强调度
- 不要比主任务卡更突出

## 7.2 闪记弹层

### 闪记目标

- 成为全产品最快的录入层
- 输入后立刻结束，不让用户停留

### 闪记布局

```text
顶部标题
输入框
语音入口
快捷提醒时间
快捷标签
底部操作按钮
```

### 闪记视觉重点

焦点顺序建议为：

1. 输入框
2. 快捷提醒
3. 保存按钮

### 关键设计细节

#### 输入框

- 默认展开 2-3 行高度
- 占据弹层视觉中心
- 占位文案尽量自然，如“把刚刚想到的内容记下来”

#### 快捷提醒

- 采用胶囊按钮横向排列
- 默认推荐项可带轻微高亮
- 点击后立即体现选中状态

#### 保存按钮

- 文案建议：`保存并回去`
- 必须固定在底部，保证单手易点

### 动效建议

- 弹层上滑时长控制在 200-250ms
- 保存后用轻 toast 或状态条反馈
- 不做复杂成功动画

## 7.3 回看箱

### 回看箱目标

- 让用户快速清理到期记录
- 让列表更像“回收箱”而不是“任务管理器”

### 回看箱布局

```text
顶部标题
分段筛选
到期提示条
记录列表
底部批量操作
```

### 记录卡片 UI

卡片建议分为 3 层信息：

1. 主信息层

- 记录内容
- 类型标签

1. 上下文层

- 来源任务
- 记录时间
- 提醒时间

1. 操作层

- 已处理
- 转待办
- 延后
- 归档

### 操作优先级

默认最突出按钮建议为：

- `已处理`

原因：

- 它最符合“快速清空挂起事项”的主要目标

### 空状态设计

空状态要给用户“收拢完成”的感受。

建议包含：

- 简洁图形
- 完成文案
- 次要行动，如 `返回首页继续学习`

## 7.4 统计页

### 统计页目标

- 用最少的图表让用户看见规律
- 不做复杂 BI 风格

### 统计页布局

```text
时间维度切换：今日 / 本周
核心数字卡片
类型分布图
时间段分布图
轻量建议卡片
```

### 图表建议

- 类型分布：环图或横向条形图
- 时间分布：按时段柱状图
- 趋势变化：简洁折线

### 视觉原则

- 图表样式尽量简单
- 不使用过多辅助线
- 重点数字比图形更重要

## 7.5 我的页

### 我的页目标

- 容纳所有低频设置
- 不打扰主流程

### 我的页布局

```text
个人信息区
提醒设置
标签管理
数据导出
帮助与关于
```

### 样式建议

- 列表型布局即可
- 少卡片，多分组
- 强调可读性与可维护性

## 8. 关键状态设计

首版必须覆盖以下状态，否则页面会显得不完整。

### 首页状态

- 未开始学习
- 专注中
- 暂停中
- 今日无待回看
- 今日有待回看

### 闪记弹层状态

- 初始空白
- 正在输入
- 已选提醒
- 语音识别中
- 保存成功
- 保存失败

### 回看箱状态

- 列表有到期项
- 列表为空
- 正在延后
- 已处理完成

### 统计页状态

- 数据为空
- 今日数据
- 本周数据

## 9. 交互反馈规范

反馈要轻，但必须明确。

### 成功反馈

- 使用轻 toast 或顶部状态条
- 文案示例：`已记下，稍后提醒你`

### 风险提醒

- 通知未开启：提示但不强迫
- 麦克风未授权：给替代路径，不中断输入

### 删除与归档

- 首版建议优先用“归档”代替删除
- 删除如果存在，应做二次确认

## 10. 无障碍与易用性建议

- 文本与背景对比度保持可读
- 重要操作避免只用颜色区分
- 支持较大字号时不破坏主流程
- 语音入口要有明确图标与文字

## 11. 首版设计优先级

如果设计资源有限，建议按以下顺序出高保真稿：

1. 闪记弹层
2. 首页
3. 回看箱
4. 统计页
5. 我的页

原因：

- 闪记弹层决定核心价值是否成立
- 首页决定主路径是否顺畅
- 回看箱决定闭环是否完成

## 12. 交付建议

如果继续推进设计执行，下一步最适合继续产出：

- 低保真线框图
- 中保真页面稿
- 组件清单
- 设计 token 表
- 交互说明稿
